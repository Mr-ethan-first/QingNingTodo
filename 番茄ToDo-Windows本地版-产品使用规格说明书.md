## 二、产品概述

番茄 ToDo 是一款基于「番茄工作法（Pomodoro Technique）」设计的时间管理与专注效率工具，核心理念为 **规划 → 追踪 → 记录 → 处理 → 可视化**。

本 Windows 本地版对原产品进行 1:1 功能复刻，核心定位为：

> **简约高效的「番茄钟 + 待办列表 + 数据统计」桌面工具，无广告、秒启动、纯本地存储。**

产品三大核心模块：

1. **待办列表**：规划每一天的任务，让一天在计划中有条不紊。
2. **番茄钟计时**：以 25 分钟工作 + 5 分钟休息为基础的可自定义专注计时。
3. **数据统计**：以图形化方式直观展示专注成果，提供成就感与自我复盘依据。

设计原则：极简风格，界面简洁美观，操作逻辑清晰，学习成本趋近于 0，操作过程以流畅动画衔接。

## 四、详细功能规格

### 4.1 待办清单（ToDoList）

#### 4.1.1 功能描述
待办清单是产品入口页，展示用户的所有待办事项卡片，与番茄钟深度结合，点击待办即可发起专注计时。

#### 4.1.2 功能明细

| 编号 | 功能点 | 规格说明 |
| --- | --- | --- |
| T-01 | 新建待办 | 主界面右上角「+」新增待办，输入待办名称 |
| T-02 | 待办集（分类） | 可创建待办集，将大事项细分为多个子任务；每个待办集可选颜色标记以区分不同分类/项目 |
| T-03 | 计时方式选择 | 新建时选择计时方式：普通番茄钟（默认 25 分钟工作 + 5 分钟休息）/ 正计时 / 倒计时 / 严格模式 |
| T-04 | 时长设置 | 可自定义计时时长（如预设 5 / 25 / 35 分钟，或自定义） |
| T-05 | 重复选项 | 支持设置重复：每日 / 每周 / 每月 周期性任务 |
| T-06 | 到点提醒 | 可为待办设置到点提醒（本地系统通知） |
| T-07 | 背景图 | 可为待办事项选择背景图，制定个性化卡片 |
| T-08 | 编辑待办 | 长按（桌面端为右键 / 悬停操作）待办事项进行编辑 |
| T-09 | 开始专注 | 点击待办事项右侧「开始」按钮进入番茄时间 |
| T-10 | 完成与勾选 | 完成后可勾选；普通番茄钟一次性任务完成后自动销毁（不再系统提示） |
| T-11 | 优先级 | 任务可按优先级排入「To Do Today」列表，预估每个任务工作量 |
| T-12 | 待办排序 | 支持在列表中调整待办展示顺序 |

#### 4.1.3 计时方式定义

| 计时方式 | 定义 |
| --- | --- |
| 普通番茄钟 | 25 分钟工作 + 5 分钟休息，休息结束有振动/提示；适合安排一天学习计划 |
| 正计时 | 从 0 开始累计，手动结束，适合不定时长任务 |
| 倒计时 | 设定固定时长倒数计时 |
| 严格模式 | 计时开启后不能暂停，提前完成可取消；用于强约束专注 |

### 4.2 番茄钟（专注计时核心）

#### 4.2.1 功能描述
番茄钟是产品核心计时引擎，进入专注界面后开始计时，计时结束进入休息模式，可循环往复。

#### 4.2.2 专注界面控件

| 编号 | 控件 | 规格说明 |
| --- | --- | --- |
| P-01 | 屏幕/窗口常亮 | 计时期间保持窗口/屏幕常亮，不息屏 |
| P-02 | 暂停 | 暂停当前计时（严格模式下不可用） |
| P-03 | 任务循环 | 设定番茄钟工作/休息的循环次数，自动循环 |
| P-04 | 放弃本次计时 | 放弃当前番茄钟（不计入有效专注） |
| P-05 | 提取/结束本次计时 | 结束并保存本次专注记录（区别于「放弃」） |
| P-06 | 定目标 | 计时中可点击中间「定目标」，写下长期目标与截止时间，主界面持续显示 |
| P-07 | 专注悬浮窗 | 计时页面右上角开启，桌面悬浮显示计时进度（窗口置顶小窗） |

#### 4.2.3 休息模式

| 编号 | 功能点 | 规格说明 |

#### 4.2.4 番茄钟高级特性

| 编号 | 功能点 | 规格说明 |
| --- | --- | --- |

### 4.3 专注模式（原「学霸模式」，Windows 适配）

#### 4.3.1 功能描述
原「学霸模式」在移动端于专注期间锁定其他娱乐 App、拦截消息。Windows 本地版将其适配为「桌面专注模式」，专注计时期间限制/拦截干扰源应用与通知。

#### 4.3.2 功能明细

| 编号 | 功能点 | 规格说明 |
| --- | --- | --- |
| F-01 | 专注锁定 | 专注计时期间，检测并阻止/警告打开被列为干扰源的应用（娱乐类程序） |
| F-02 | 分心警告 | 用户尝试切换到非白名单应用时发出警告提示 |
| F-03 | 通知拦截 | 计时期间拦截系统与第三方应用弹窗/消息通知，避免打扰 |
| F-04 | 白名单 | 可添加允许例外的应用（工作必需程序）；添加白名单时支持搜索查找 |
| F-05 | 严格模式联动 | 严格模式下强化拦截，计时不可暂停，中途无法轻易退出 |

> 平台适配说明：Windows 端通过前台窗口监控 + 通知策略（勿扰模式/专注助手）+ 进程管控实现，需在应用内说明并获取用户授权；不涉及对系统底层的破坏性操作。

## 五、数据库设计（MySQL）

### 5.1 设计说明
- 数据库引擎：InnoDB，字符集 `utf8mb4`，排序规则 `utf8mb4_general_ci`。
- 本地单用户模式，默认单用户；保留 `user` 表以支持多本地档案（可选）。
- 所有时间字段使用 `DATETIME`，时长使用秒（`INT`）为单位。

### 5.2 表结构

#### 5.2.1 用户/本地档案表 `user`
| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| nickname | VARCHAR(64) | NOT NULL | 昵称 |
| avatar_path | VARCHAR(255) | NULL | 头像本地路径 |
| created_at | DATETIME | NOT NULL | 创建时间 |
| updated_at | DATETIME | NOT NULL | 更新时间 |

#### 5.2.2 待办集表 `todo_group`
| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| user_id | BIGINT | FK → user.id | 所属用户 |
| name | VARCHAR(128) | NOT NULL | 待办集名称 |
| color | VARCHAR(16) | NULL | 颜色标记（HEX） |
| sort_order | INT | DEFAULT 0 | 排序 |
| created_at | DATETIME | NOT NULL | 创建时间 |

## 七、非功能性需求

| 类别 | 要求 |
| --- | --- |
| 性能 | 秒启动；计时误差 ≤ 1 秒/25 分钟；界面动画流畅（≥ 60fps 体验目标） |
| 无广告 | 全程无广告 |
| 离线可用 | 100% 本地运行，无需联网 |
| 数据安全 | 本地 MySQL 存储，支持备份/恢复；敏感操作二次确认 |
| 兼容性 | Windows 10 / 11 x64 |
| 可用性 | 学习成本趋近 0，操作逻辑清晰 |
| 通知 | 使用 Windows 原生通知实现到点提醒、休息提示 |
| 权限合规 | 专注模式的进程/通知管控需明确告知并获用户授权，不做破坏性系统操作 |

| 移动端功能 | Windows 本地版适配方案 |
